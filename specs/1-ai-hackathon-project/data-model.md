# Data Model Design: AI-Driven Hackathon Project

## Spec-Engine Data Model

### TOML Configuration Structure
```
BookSpec {
  title: string
  description: string
  chapters: ChapterSpec[]
}

ChapterSpec {
  id: string
  title: string
  description: string
  frontmatter: object
}
```

### Generated Content Structure
```
GeneratedChapter {
  id: string
  title: string
  content: string
  frontmatter: object
  filePath: string
}
```

## RAG-Backend Data Model

### Document Collection
```
Document {
  id: string
  content: string
  metadata: object
  embedding: float[]
}
```

### Query Request/Response
```
QueryRequest {
  query: string
  top_k: int = 5
}

QueryResponse {
  answer: string
  sources: Document[]
}
```

## Relationships Between Models

The Document model in the RAG-Backend will be populated with content generated by the Spec-Engine based on the BookSpec and ChapterSpec models. The GeneratedChapter content will be transformed into Document objects for vector storage.
